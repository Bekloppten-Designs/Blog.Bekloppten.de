Eigener Webserver/Website mit Nginx

Für den Hier Beschriebenen Weg Brauchst du bestenfalls ein kostenloses Konto bei Cloudflare und eine Domain. Diese Bekommst du bei Anbietern wie Netcup für 5€ im Jahr.
Schaue dir auf jeden Fall vorher die Cloudflare Tunnel Anleitung an!
Außerdem Brauchst du Docker

Mit NGINX und Docker kannst du schnell statische Webseiten hosten. Jede Webseite ist einfach ein Ordner mit HTML-Dateien.

🚀 1. Eine einzelne Webseite

📁 Ordner erstellen

⚠️ Bitte überprüfe Befehle vor dem Ausführen. Ich übernehme keine Haftung. 


mkdir -p /Docker/Nginx/site

Du kannst site natürlich nenne wie du willst

🧾 HTML Datei erstellen 


nano /Docker/Nginx/site/index.html

Inhalt


läuft

Danach Speichern:
STRG + O
ENTER
STRG + X

⚙️ Docker Compose anlegen 


nano /Docker/Nginx/compose.yaml

Inhalt:


services:
  nginx:
    image: nginx:latest
    container_name: nginx_site
    ports:
      - "8080:80"
    volumes:
      - ./site:/usr/share/nginx/html:ro
    restart: unless-stopped

▶️ Starten 


cd /Docker/Nginx
docker compose up -d

Wenn eine Fehlermeldung kommt, trage in der Compose einen anderen Port ein

➡️ Öffnen:

http://DEINE_IP:8080
Jetzt kannst du mit Cloudflare Die Seite öffentlich machen!

🌍 2. Mehrere Webseiten

📁 Struktur


/Docker/Nginx
 ├── site1/
 ├── site2/
 └── site3/

⚙️ Docker Compose 


services:
  site1:
    image: nginx:latest
    container_name: site1
    ports:
      - "8081:80"
    volumes:
      - ./site1:/usr/share/nginx/html:ro
    restart: unless-stopped

  site2:
    image: nginx:latest
    container_name: site2
    ports:
      - "8082:80"
    volumes:
      - ./site2:/usr/share/nginx/html:ro
    restart: unless-stopped

  site3:
    image: nginx:latest
    container_name: site3
    ports:
      - "8083:80"
    volumes:
      - ./site3:/usr/share/nginx/html:ro
    restart: unless-stopped

Dies kannst du natürlich Beliebig oft wiederholen für beliebig viele Seiten.
Füge wie oben jetzt zum Prüfen deine Test Daten ein.
Du kannst jetzt auch wie oben jede seite mit Cloudflare Tunneln

🔄 Automatische Git‑Updates für Webseiten aus GitHub

Wenn deine Webseiten in einem GitHub‑Repository liegen, kannst du sie automatisch aktuell halten.
Dazu wird das Repo einmal in den jeweiligen Ordner geklont und danach regelmäßig per Skript aktualisiert.

1️⃣ Git Installieren Git‑Repository in die Website‑Ordner klonen 


sudo apt update && sudo apt install git -y

Beispiel für drei Seiten:

⚠️ Entferne alle Testdaten aus den Ordnern, bevor du den Nächten schritt machst 


cd /Docker/Nginx/site1
git clone https://github.com/dein-user/dein-repo-site1.git .


cd /Docker/Nginx/site2
git clone https://github.com/dein-user/dein-repo-site2.git .


cd /Docker/Nginx/site3
git clone https://github.com/dein-user/dein-repo-site3.git .

Wichtig: Der Punkt . am Ende sorgt dafür, dass die Dateien direkt in site1, site2, site3 landen.

2️⃣ Skript‑Ordner und Update‑Skripte

Lege einen Skript‑Ordner an: 


mkdir -p /Docker/Nginx/scripts

Struktur: 


/Docker/Nginx/
 ├── site1/
 ├── site2/
 ├── site3/
 ├── compose.yaml
 └── scripts/
      ├── update_site1.sh
      ├── update_site2.sh
      └── update_site3.sh

Nun legen wir die Scripts an


nano /Docker/Nginx/scripts/update_site1.sh


#!/bin/bash
cd /Docker/Nginx/site1 || exit
git pull

Danach Speichern:
STRG + O
ENTER
STRG + X

Mach das jetzt für alle deine Seiten

Ausführbar machen:


chmod +x /Docker/Nginx/scripts/*.sh

3️⃣ Cronjob: Automatische Updates alle 5 Minuten

Cron öffnen:


crontab -e
Eintragen: 

*/5 * * * * /Docker/Nginx/scripts/update_site1.sh
*/5 * * * * /Docker/Nginx/scripts/update_site2.sh
*/5 * * * * /Docker/Nginx/scripts/update_site3.sh

Die 5 Steht für die minuten.
Ergänze hier alle deine Seiten.
Immer wenn du jetzt auf Github ein änderung machst, wird sie direkt auf deinem server übernommen
